# D — Defense | I — Infrastructures | 07 — Cyber Range and Information Operations Infrastructure
## AMPEL360 MRTT Q300 (MRTT-Q300)

## 1. Purpose

This folder assesses **cyber range and information operations infrastructure** for the AMPEL360 MRTT Q300 programme. It covers the cyber resilience testing environment, red-team/blue-team infrastructure, avionics cybersecurity testing, and information operations (IO) assurance infrastructure relevant to the MRTT-Q300 platform.

## 2. Scope

- Cyber range infrastructure for MRTT-Q300 avionics and mission system cyber testing
- Red-team / penetration testing infrastructure for avionics networks
- IFF/Mode 5 and data link (Link 16/JREAP) security testing infrastructure
- Electronic warfare (EW) resilience test infrastructure
- Secure software update and patch management infrastructure
- Information environment monitoring and situational awareness infrastructure

## 3. Key Assessment Areas

| Assessment Area | Primary Question |
|----------------|-----------------|
| Avionics cyber range | Is a dedicated cyber range available for MRTT-Q300 avionics? |
| Red-team capability | Is adversarial testing infrastructure in place for mission systems? |
| IFF/datalink assurance | Can IFF/Link 16 security be tested in a controlled environment? |
| EW resilience | Is EW/jamming resilience test infrastructure available? |
| Patch management | Is secure OTA/ground-based patch infrastructure defined? |
